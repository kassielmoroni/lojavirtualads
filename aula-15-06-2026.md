# Etapa 2 – Configurar o Ambiente Docker

## baixar o docker desktop
instalar para PERSONAL (não precisa admin)
depois rodar wsl --install no terminal para habilitar virtualização

se der erro 
 wsl --shutdown 
 wsl --update
 wsl -l -v --all

Se listar as distros, reinicie o Docker Desktop.
Se ainda travar, reinicie o Windows e rode:

wsl -l -v --all

Depois abra o Docker Desktop novamente.

## Criar os arquivos Docker

Crie na raiz do projeto:

```text
Dockerfile
docker-compose.yml
```

Baixar arquivos do Moodle.


## Criar o arquivo .env

```bash
copy .env.example .env
```

## Configurar o banco de dados

```env
DB_CONNECTION=mysql
DB_HOST=db
DB_PORT=3306
DB_DATABASE=lojavirtual
DB_USERNAME=root
DB_PASSWORD=root
```

## Instalar dependências

```bash
composer install
npm install
npm run build
```

## Subir os containers

```bash
docker compose up -d --build
docker ps
```

## Gerar chave da aplicação

```bash
docker compose exec app php artisan key:generate
```

## Executar migrations

```bash
docker compose exec app php artisan migrate
```

## Testar a aplicação

Acesse:

Aplicação
http://localhost:8000

PhpMyAdmin
http://localhost:8080

# Etapa 3 – Entendendo a Proteção da Branch Main

Fluxo utilizado:

```text
feature/minha-alteracao
           │
           ▼
     Pull Request
           │
           ▼
         main
```

## Testando a proteção

```bash
git add .
git commit -m "Teste de proteção"
git push origin main
```

Observe a mensagem de erro retornada pelo GitHub.

# Etapa 7 – Criar uma Pull Request

1. Acesse o GitHub.
2. Clique em **Compare & Pull Request**.
3. Base Branch: `main`.
4. Compare Branch: sua branch.
5. Informe título e descrição.
6. Clique em **Create Pull Request**.
